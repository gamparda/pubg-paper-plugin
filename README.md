# Battlegrounds

Paper 26.2(Java 25)용 PUBG 스타일 아이템·전투 시스템 플러그인이다. 맵이나 자기장 생성기는 포함하지 않고, 맵이 없어도 개발·시험할 수 있는 아이템과 플레이어 상태를 우선 구현한다.

## 구현된 시스템

### 전투

- 첨부 목록 전체 92종: AR/DMR/SR/SMG/SG/LMG/권총/석궁/중화기/근접/투척
- PDC 기반 14종 탄약
- 우클릭 사격, 좌클릭 재장전
- 블록 충돌, 거리 감쇠, 머리/몸통/사지 판정
- PUBG 100 HP와 Minecraft 20 HP 사이의 `health-scale`
- 프라이팬 보유 시 후방 하체 탄환 방어
- 단발/점사/연사 모드 전환(웅크린 채 좌클릭)
- 단발은 우클릭을 놓았다가 다시 눌러야 재발사; 연사는 우클릭을 누르는 동안 무기별 RPM으로 서버 스케줄링
- 산탄총 다중 펠릿·분산도
- 수류탄/연막/섬광/화염병/블루존 수류탄/중화기 투사체 효과
- 각 무기별 원문 URL, PC 성능표 추출값 또는 fallback 출처 표시

### 방어구

- PC 기준 1/2/3레벨 헬멧: 30%/40%/55%, 내구도 100
- PC 기준 1/2/3레벨 조끼: 30%/40%/55%, 내구도 100
- 헬멧은 내구도 0에서 소멸
- 조끼는 내구도 0에서 남으며 피해 감소율 20% 유지
- 조끼 소지 용량 +50
- 올인원 수리 키트로 장착 방어구 수리

### 소지 용량과 배낭

- 기본 용량 50
- 1/2/3레벨 배낭 +150/+200/+250
- 전파 방해 배낭 +200 및 블루존 피해 흡수 에너지
- PDC 아이템 무게를 계산해 용량 초과 아이템 줍기 차단

### 치료·부스트

- 붕대: 3초, 12 HP, 75 HP 제한
- 구급상자: 6초, 75 HP까지
- 의료용 키트: 6초, 100 HP까지
- 에너지 드링크: 4초, 부스트 +40
- 진통제: 6초, 부스트 +60
- 아드레날린: 6초, 부스트 +100
- 부스트 30초당 10 감소
- 단계별 8초당 체력 회복 및 60%/90% 구간 속도 증가
- 소모품 사용 중 진행률 표시
- 설정 거리 초과 이동, 피격, 손 아이템 변경 시 사용 취소

### 기절·소생

- 팀원이 살아 있거나 자가제세동기를 보유한 상태의 치명 피해는 DBNO 전환
- 출혈 제한시간 후 사망
- 팀원 우클릭 소생
- 응급 지혈 주사 1초 소생
- 자가제세동기 10초 자가 소생
- 자가제세동기는 사용 시작 시 소비되며 중단되어도 반환되지 않음
- 사망 시 블루칩 드롭

### 기타 장비

설정 및 PDC 아이템으로 등록된 장비:

- 상시 낙하산: 낙하 중 우클릭으로 전개
- 녹지/설원/사막 길리 슈트
- 전술 가방
- 응급 처치 장비: 치료제 사용시간 단축
- 전투 준비 키트: 체력·부스트 완전 회복
- 접이식 방패: 1.5초 설치 후 2블록 철창 엄폐물 생성
- 제리캔
- 비상 호출
- 블루칩 송신기
- 블루존 유도기

제리캔의 차량 급유, 비상 호출의 안전지대 중심 이동, 블루칩 송신 부활 비행, 블루존 유도기의 다음 원 변경은 각각 차량·매치·블루존 모듈이 생겨야 의미가 있으므로 현재는 **아이템 정의와 연동 지점**까지만 제공한다.

## 명령어

```text
/bg give <ID/이름/별칭> [수량]
/bg find <검색어>
/bg reload
/bg status
/bg vehicle list
/bg vehicle give <차량-id> [수량]
/bg vehicle spawn <차량-id>
```

## 이동수단

16종 이동수단과 서버 주도 물리를 지원한다: 모터사이클, 스쿠터, 툭샤이, 산악 자전거, 스노우 모빌, 다시아, 미라도, 포니 쿠페, 미니 버스, UAZ, 버기, BRDM-2, 모터글라이더, 보트, 아쿠아레일, 고무보트.

- 전진/후진: W/S
- 조향: A/D
- 부스트: 전력질주
- 핸드브레이크: 웅크리기
- 모터글라이더 상승/하강: 전력질주/웅크리기
- 차량별 최고속도, 후진속도, 가속력, 회전력, 좌석, 연료, 내구도
- 육상/수상/수륙양용/공중 지형별 물리
- 고속 하차 피해, 로드킬, 피격 내구도, 파괴 5초 후 폭발
- 제리캔 급유와 올인원 수리 키트 연동
- BRDM 팀 잠금

전체 설정표: [`docs/03-vehicle-catalog.md`](docs/03-vehicle-catalog.md)

## 탄도와 파티클

총기는 즉시 명중하는 직선 hitscan 대신 1틱 단위 탄환 궤적을 사용한다.

- 무기별 `bullet-speed-mps`, `spread-degrees`, `gravity-scale`, `drag-per-second`
- 중력 낙차와 공기저항
- 각 틱 이동 구간 전체 ray trace로 고속 탄환의 벽·엔티티 관통 방지
- 노란 탄도 트레이서와 25블록 간격의 붉은 낙차 표식
- 총구 화염·총구 연기·벽 충돌 연기·명중 파티클
- 산탄총은 모든 펠릿에 탄도를 적용하고 기본 3개 펠릿의 궤적을 표시
- 최대 활성 탄환 제한 초과 시 기존 ray trace로 폴백

```yaml
ballistics:
  enabled: true
  gravity-mps2: 9.81
  drag-per-second: 0.01
  trail-spacing-blocks: 2.0
  drop-marker-spacing-blocks: 25.0
  shotgun-tracer-pellets: 3
  max-active-bullets: 500
```

권한: `battlegrounds.admin`

모든 ID는 `/bg give` 탭 완성에 나타난다. 정확한 ID 외에도 한글 이름·영문 별칭·부분검색을 지원하며 수량은 1~64로 제한한다.

```text
/bg give akm 5
/bg give 베릴 1
/bg give bandage 10
/bg find grenade
```

전체 무기 수치와 출처 링크는 [`docs/02-weapon-catalog.md`](docs/02-weapon-catalog.md)에 정리되어 있다.

## 빌드

```bash
mvn clean package
```

- 요구 런타임: Java 25
- 대상 API: `io.papermc.paper:paper-api:26.2.build.56-alpha`
- 결과물: `target/battlegrounds-0.5.0.jar`

## 테스트

순수 도메인 테스트는 다음을 검증한다.

- 거리·부위·방어구 피해 계산
- 탄창 소비와 재장전
- DBNO·사망·회복·소생
- 방어구 파괴와 파괴 조끼 20% 방호
- 기본+조끼+배낭 용량 계산
- 시간제 행동 완료와 취소
- 부스트 감소·회복·속도 구간
- 전파 방해 배낭, 접이식 방패, 자가제세동기 상태

실서버 런타임 시험은 별도 테스트 서버에서 수행해야 한다.
